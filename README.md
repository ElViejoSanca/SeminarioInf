# ¡Bienvenidos!



### Este repositorio está pensado para la entrega incremental de los trabajos del seminario de informática.



**Desde la fecha 28/6/2026 este repositorio probablemente no recibirá futuras actualizaciones, debido a que es la última entrega de este proyecto.**



Ya se encuentra disponible la primer release del proyecto, con todo el código fuente para las clases java correspondientes y la creación de la DB utilizada en conjunto para la permanencia de datos.



El proyecto en cuestión: SIGIOST (Sistema de Gestión de Ordenes de Servicio Técnico).

Un programa pensado, como su nombre lo indica, para la creación, gestión y auditoría de Ordenes de Servicio Técnico dentro de una empresa de ISP.

Este fue pensado como una solución integradora a un proceso que actualmente se realiza de manera distribuida entre varias aplicaciones, sitios y sistemas que no siempre funcionan en conjunto de manera ideal o totalmente automatizada como en este caso.



Características principales:

\-Permanencia de datos permitiendo administración de usuarios, sucursales, técnicos, tickets, ítems, etc.

\-Creación de tickets de manera intuitiva por operadores de CX.

\-Conversión de tickets en Ordenes de servicio por operadores de Coordinación y Asistencia Tecnica.

\-Asignación de técnicos según carga actual, ubicación relativa y especialización.

\-Administración del stock de equipos y material disponible con actualización concreta.

\-Capacidad para auditar ordenes según cierre, tiempo, y material utilizado.

\-Flexibilidad para futura implementación de mas características, cambios de base de datos y escalabilidad general del proyecto.
